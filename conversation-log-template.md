# Conversation log — Human ↔ CampVault Agent

> **用途**：填入 Devfolio 提交的 `conversationLog` 字段（全文或公开文件链接）。  
> **原则**：按时间或主题记真实对话摘要；可略去敏感信息（密钥、具体钱包地址可写「0x…已脱敏」）。

---

## 1. 项目启动与目标对齐

- **日期**：
- **人类**：最初想解决什么问题？是否绑定 Uniswap / 押金 / 打卡等？
- **Agent**：如何理解需求、提出的首个技术路径？
- **结论**：范围边界（MVP 含什么、不含什么）。

---

## 2. 架构与链选型

- **讨论点**：为何选 Base、合约分层、是否用 miniapp。
- **Agent 输出**：草图、模块列表、风险点。
- **人类决策**：采纳 / 调整点。

---

## 3. 迭代：核心功能开发

（每一大功能可复制一小节）

### 3.x （功能名，如：押金合约 / 打卡逻辑）

- **日期**：
- **人类**：需求或验收标准。
- **Agent**：实现思路、文件变更摘要。
- **卡点**：bug / 不确定处；如何一起解决。

---

## 4. Uniswap / Skills / Agent 工具集成（若适用）

- **人类**：希望 Agent 或用户完成什么链上/链下动作？
- **Agent**：读了哪些文档（可对应 `helpfulResources`）、写了哪些 skill 或封装？
- **结果**：成功路径 + 已知限制。

---

## 5. 测试、部署与 Demo

- **测试网 / 主网**部署记录（人类确认环境）。
- **Agent**：部署步骤文档化、环境变量清单（不含密钥）。
- **Demo / 视频**：谁录、Agent 是否写了脚本或讲稿要点。

---

## 6. 提交与元数据

- **track 选择**：人类与 Agent 如何选定赛道。
- **submissionMetadata**：`skills`、`tools` 列表是如何从真实使用反推的（一两段说明即可）。
- **Self-custody 转移**：人类完成钱包转移的时间点（不涉及私钥细节）。

---

## 7. 未解决 / 赛后计划

- 已知 issue、技术债。
- `intention`（continuing / exploring / one-time）对应的讨论摘要。

---
